HTML结构更新

HTML结构改进：

<!-- 项目卡片容器 --> <div v-else-if="upcomingProjects.length > 0" class="projects-grid"> <article v-for="(project, index) in upcomingProjects" :key="project.code" class="listing-card" > <!-- 卡片内容 --> </article> </div>

主要变化：

  • 添加了 projects-grid 容器包装所有卡片
  • 保持了原有的 listing-card 样式
  • 条件渲染逻辑保持不变
响应式CSS实现

响应式断点设计：

桌面端（>768px）：
.projects-grid { display: grid; grid-template-columns: repeat(2, 1fr); gap: 24px; max-width: 1000px; }
平板端（≤768px）：
@media (max-width: 768px) { .projects-grid { grid-template-columns: 1fr; gap: 20px; max-width: 500px; } }
手机端（≤480px）：
@media (max-width: 480px) { .projects-grid { gap: 16px; } }
视觉效果优化

布局优势：

  • 空间利用：更好地利用宽屏空间，避免卡片过宽
  • 视觉平衡：两个卡片并排显示，视觉更加平衡
  • 信息密度：在同一视窗内展示更多项目信息
  • 用户体验：减少滚动，提高浏览效率
📱 移动端适配

在移动设备上，布局会自动切换为单列，确保在小屏幕上的可读性和可用性。每个卡片仍然保持完整的信息展示和交互功能。
